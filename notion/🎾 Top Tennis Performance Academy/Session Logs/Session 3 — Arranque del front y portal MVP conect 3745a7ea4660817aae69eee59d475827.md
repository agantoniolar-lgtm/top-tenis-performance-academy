# Session 3 — Arranque del front y portal MVP conectado a Supabase

Date: June 2, 2026
Key decisions: 1. Portal MVP solo muestra lo conectado a Supabase — no hay features vacíos. 2. Sidebar limpio: solo Atletas y Nuevo Reporte hasta que existan más features reales. 3. PTF / Athlete Voice es funcionalidad del atleta, no del coach; el coach solo desbloquea. 4. Physical se llena opcionalmente cada 3 meses dentro del mismo NuevoReporte. 5. Queries separadas por sección (no nested PostgREST) para evitar conflictos con palabras reservadas. 6. Expediente.jsx con 8 tabs queda en el repo como referencia de diseño futuro, no en routing activo.
Open items / follow-ups: 1. Backfill del campo 'return' de on-court (excluido del select por conflicto PostgREST — considerar renombrar la columna a 'resto' en BD). 2. Athlete Voice — flujo para que el atleta llene su autoevaluación después de que coach desbloquee. 3. Borrar datos de seed cuando se tengan datos reales. 4. Physical section — validar formato con el equipo (frecuencia trimestral, campos opcionales). 5. Pantalla del coach con listado de reportes pendientes por periodo. 6. Investigar si renombrar columna 'return' → 'resto' en report_on_court para evitar workarounds.
Status: Complete
What we did: Procesamos el design handoff de Claude Design (Portal - Expediente del Atleta) e implementamos el front completo. Primero instalamos el sistema de diseño: variables CSS, fuentes (Bricolage Grotesque, Manrope, Big Shoulders Display), primitivos UI (Icon, Card, Btn, Tag, ScoreRing, etc.), y el shell del portal con Sidebar, Topbar y PortalLayout. Luego construimos la vista completa del Expediente con 8 tabs (Resumen, On-court, Physical, Mental, Character, Voice, Torneos, Documentos) usando datos mock del diseño. Después ajustamos el enfoque: limpiamos el portal a solo lo conectado a Supabase real. Migramos el auth de localStorage fake a Supabase Auth (signInWithPassword), creamos el cliente Supabase, sembramos 1 coach y 5 atletas en producción, y limpiamos el sidebar a solo Atletas y Nuevo Reporte. Conectamos la lista de atletas (query real a tabla athletes con RLS), el perfil del atleta con métricas headline y grid de strokes con deltas vs mes anterior, y el NuevoReporte con 3 tabs (On-court, Physical, Character) que guardan en Supabase. Resolvimos varios bugs: faltaban registros en auth.identities, confirmation_token era NULL, y el nested select de PostgREST fallaba silenciosamente por la columna llamada 'return' (palabra reservada). Guardamos el design handoff completo en Top Tennis Performance Academy/Diseño · Portal/.
